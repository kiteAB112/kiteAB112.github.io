## 问题

个人技术主页很容易在设计主题、评论系统、搜索和数据统计中失去重心，最后迟迟没有第一篇内容。

## 这次的选择

这一版只保留 Astro、Markdown、GitHub Pages 与 GitHub Actions。写作材料和网站代码都在同一个 Git 仓库中，新增一篇内容不需要后台或数据库。

## 接下来

当内容累计到足够多，再根据实际检索和阅读需求评估标签页、RSS 或全文搜索，而不是提前为它们增加维护负担。
